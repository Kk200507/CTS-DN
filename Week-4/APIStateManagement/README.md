# API Integration & State Management

In modern single page applications (SPAs), the frontend communicates with the backend asynchronously using web APIs. The returned data must then be stored and shared across multiple views using a client-side state store.

## Contents

### 1. [api_integration.js](file:///c:/Personal/Projects/CTS-DN/Week-4/APIStateManagement/api_integration.js)
- **Topic**: Interacting with REST APIs in JavaScript.
- **Concepts**: Fetch API, Axios client configuration, global request/response interceptors, and client-side HTTP error handling.
- **Goal**: Master executing asynchronous network queries and managing failures gracefully without crashing the UI.

### 2. [state_management.md](file:///c:/Personal/Projects/CTS-DN/Week-4/APIStateManagement/state_management.md)
- **Topic**: Managing global application state.
- **Concepts**: State architectures, Redux patterns (Actions, Reducers, Selectors, Store), NgRx (Angular RxJS pipelines), and Pinia (Vue's lightweight stores).
- **Goal**: Understand how state containers work, when to use local component state vs. global state, and how to compare different framework store layouts.
