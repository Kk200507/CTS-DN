# Web Design Standards

This module focuses on the standards that define user experience, access, and compatibility: **Responsive Web Design**, **Web Accessibility (a11y)**, and **Cross-Browser Compatibility**.

---

## Contents

### 1. [responsive_design.html](file:///c:/Personal/Projects/CTS-DN/Week-4/WebDesignStandards/responsive_design.html)
- **Topic**: Modern layout scaling across devices.
- **Concepts**: Mobile First, Media Queries, fluid dimensions (`%`, `fr`), viewport units (`vw`, `vh`), Flexbox/Grid breakpoints, and modern **Container Queries** (`@container`).
- **Goal**: Build interfaces that adjust to screens ranging from watches to 4K monitors.

### 2. [web_accessibility.md](file:///c:/Personal/Projects/CTS-DN/Week-4/WebDesignStandards/web_accessibility.md)
- **Topic**: Web Accessibility (a11y) rules.
- **Concepts**: WCAG 2.1 compliance (POUR), Semantic structure landmarks, ARIA markup (`role`, `aria-*`), keyboard focus management (`tabindex`), screen reader support, and color contrast standards.
- **Goal**: Ensure applications are fully usable by individuals with disabilities (visual, motor, auditory, cognitive).

### 3. [cross_browser_compatibility.md](file:///c:/Personal/Projects/CTS-DN/Week-4/WebDesignStandards/cross_browser_compatibility.md)
- **Topic**: Developing for a fragmented browser landscape.
- **Concepts**: Layout engines (Blink, WebKit, Gecko), progressive enhancement vs. graceful degradation, feature detection (`@supports`, JS check), CSS vendor prefixes, and polyfill scripts.
- **Goal**: Deliver a reliable and functional experience on Chrome, Safari, Firefox, Edge, and older legacy browsers.
